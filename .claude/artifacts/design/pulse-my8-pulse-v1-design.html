Overview

Pulse v1 needs a mobile-first, touch-optimized private feed UI for high-signal recommendation cards. The visual direction is inspired by effect.institute, effect.solutions, and kitlangton.com without turning v1 into a full branding exercise.

Use shadcn components and Tailwind v4 as implementation primitives, but avoid a generic dashboard look. The interface should feel like a compact signal surface: focused, readable, tactile, and optimized for quick inspection.

Users & Goals

Seif is the only human user. Layla/agents are producer actors, not UI users — they create feed items through the protected API.

Seif's Goals

  • Open Pulse privately through Clerk authentication.
  • See unread recommendations first.
  • Quickly understand why each item matters.
  • Open the original source when interested.
  • Mark items read/unread.
  • Leave thumbs up/down feedback with optional reasoning.

User Journeys

Inspect Today's Signals

Who
Seif
Goal
See what is worth inspecting now.
Trigger
Seif opens Pulse.
Flow
Seif passes the Clerk-backed private access gate, lands on unread items sorted by recency, scans each card's title / source / TLDR / media, opens the source when useful, and marks items read as they are handled.

Give Taste Feedback

Who
Seif
Goal
Tell Layla whether a recommendation was useful.
Trigger
Seif reacts to an item.
Flow
Seif taps thumbs up or down, optionally adds a short reason, and returns to the feed without navigating through a heavy form.

Review Read Items

Who
Seif
Goal
Find previously inspected items.
Trigger
Seif switches from unread feed to read/history.
Flow
Seif opens a read/history view or filter where read items remain sorted by recency or read time.

Required States

Success
Unread feed renders cards sorted by recency; feedback and read/unread changes visibly apply.
Error
Auth failure, feed load, feedback submit, read/unread update, and source-open fallback must show clear messages. Failed per-item actions must roll back optimistic UI state.
Loading
Initial feed uses card-shaped skeletons. Per-card actions (read/unread, feedback) use optimistic updates with rollback. Pagination uses an inline indicator at the cursor boundary.
Empty
Unread feed empty state communicates that there is nothing new to inspect.
Partial
Items without media must still feel complete; optional feedback reasoning can be absent.

Visible Fields

  • Titlerequired
  • Source name or URLrequired
  • Image / media (by URL)optional
  • Body with TLDR / key inforequired
  • Created / received timerequired
  • Read / unread affordancerequired
  • Open-source actionrequired
  • Thumbs up / down (mutable)required
  • Reasoning input on feedbackoptional

Feed List Behavior

  • Unread feed and read/history view both load pages bounded by a cursor.
  • Continuation must be surfaced at the page boundary rather than rendering an unbounded list.
  • Per-card mutations (read/unread, feedback) update optimistically and roll back on failure.

Constraints

  • Mobile-first and touch-first: large hit targets, comfortable vertical rhythm, readable card density, and no hover-only interactions.
  • Private single-owner app using Clerk: no public profile, sharing, comments, likes, follows, or social graph UI.
  • No manual feed item creation UI in v1.
  • No extensive logo/identity system in v1.
  • Use shadcn components and Tailwind v4.
  • Keep UI centered on the single feed experience rather than a dashboard.
  • Media is referenced by URL only; the UI must degrade gracefully when remote media is unavailable.
  • Feed views are paginated by cursor; the UI must never assume the full history is loaded.

Accessibility

Keyboard
Interactive controls must be keyboard reachable.
Touch targets
At least 44px where practical.
Labels
Feedback controls need accessible labels distinguishing thumbs up and thumbs down.
Color
Color cannot be the only signal for read/unread or feedback state.

Related Patterns

  • Effect references — technical, editorial, high-contrast, sparse-but-distinct visual language.
  • Kit Langton reference — personal, compact, writing-forward, high-signal presentation.
  • Social / article card metaphor — concise post or article recommendation, not a task card.

Out Of Scope

Full brand identity exploration.
Multi-column desktop dashboard.
Public social profile or sharing surfaces.
Manual creation forms.
Feed analytics or curation dashboards.
Media upload, image hosting, or in-app media editing.
Archival, retention, or deletion UI for read/history items.
